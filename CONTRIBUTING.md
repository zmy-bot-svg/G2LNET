# Contributing to G2LNet

Thank you for your interest in contributing. We welcome improvements, bug fixes, and documentation updates.

## Development setup
1. Create a virtual environment.
2. Install dependencies with pip install -r requirements.txt.
3. Use config.yml to configure datasets and experiments.

## Reporting issues
- Provide a clear description of the problem and expected behavior.
- Include steps to reproduce, logs, and relevant configuration.
- Specify the OS, Python version, and GPU/driver details if applicable.

## Pull requests
- Keep changes focused and minimal.
- Include tests or reproducible scripts when applicable.
- Update documentation and CHANGELOG.md when you introduce user-facing changes.

## Code style
- Follow existing code style and naming conventions.
- Avoid large refactors unless necessary.

## Data and licensing
- Ensure any data used is properly licensed and cited.
- Do not include proprietary data or credentials.
